# 🧪 WorldCupAI — Feature Engineering Framework

This document describes the design, mathematical formulation, and implementation of the feature engineering framework in the WorldCupAI platform.

## 1. Modular Framework (`src/features/`)

Features are generated by independent, reusable classes that inherit from `BaseFeatureGenerator`. This modular design allows us to add, modify, or disable feature families without affecting other parts of the pipeline.

### Feature Families

1. **Team Strength (`team_strength.py`)**
   - **Elo Difference**: $Elo_{diff} = Elo_{home} - Elo_{away}$
   - **Elo Ratio**: $Elo_{ratio} = \frac{Elo_{home}}{Elo_{away}}$
   - **FIFA Rank Difference**: $Rank_{diff} = Rank_{home} - Rank_{away}$
   - **FIFA Rank Ratio**: $Rank_{ratio} = \frac{Rank_{home}}{Rank_{away}}$
   - **EA FC Attribute Differences**: Difference in average squad ratings (overall, attack, defense).

2. **Recent Form (`recent_form.py`)**
   - Computes rolling metrics over the last 5 and 10 matches:
     - **Win/Draw/Loss Rates**: Percentage of matches won, drawn, or lost.
     - **Goals Scored/Conceded**: Rolling average of goals.
     - **Clean Sheet Rate**: Percentage of matches with 0 goals conceded.
     - **Average Points**: Average points earned ($Win = 3, Draw = 1, Loss = 0$).

3. **Head-to-Head (`head_to_head.py`)**
   - **Cumulative Meetings**: Total matches played between the two teams prior to kickoff.
   - **H2H Wins/Losses/Draws**: Historical win/loss/draw counts.
   - **H2H Goal Difference**: Cumulative goal difference from the home team's perspective.

4. **Attack & Defence (`attack_features.py` / `defence_features.py`)**
   - **Attacking Rating**: Rolling 10-match average goals scored divided by a global baseline of 1.4.
   - **Defensive Rating**: Global baseline of 1.4 divided by rolling 10-match average goals conceded.

5. **Tournament Experience (`tournament_features.py`)**
   - **World Cup Participations**: Total World Cup tournaments played prior to the match year.
   - **World Cup Titles**: Total championships won prior to the match year.
   - **Knockout Stage Experience**: Groups passed, Round of 16, Quarterfinals, Semifinals, and Finals reached in past tournaments.

6. **Context (`context_features.py`)**
   - **Rest Days**: Days since the team's last match (clipped to a maximum of 30 days).
   - **Rest Difference**: $Rest_{diff} = Rest_{home} - Rest_{away}$.
   - **Match Flags**: `is_neutral`, `is_world_cup`, `is_friendly`.

---

## 2. Temporal Safety & Leakage Prevention

To ensure that no future information leaks into the features of past matches:
1. **Chronological Sorting**: The master dataset is sorted by `date` before feature generation.
2. **One-Match Shift**: For all rolling calculations (Recent Form, H2H, Rest Days), the series is shifted by **1 match** (`.shift(1)`) before applying the rolling window or cumulative sum.
3. **Temporal Join for Rankings**: FIFA rankings are joined using `merge_asof(direction='backward')`, ensuring that a match on date $D$ only uses the ranking published on or before date $D$.
4. **Historical Tournament Experience**: Tournament stats are joined based on the latest completed World Cup version strictly before the match year.
