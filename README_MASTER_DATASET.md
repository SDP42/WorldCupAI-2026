# 📦 WorldCupAI — Master Dataset Documentation

> Technical documentation of the final master dataset (`master_dataset.parquet`) generated by the Phase 2 ETL pipeline.

---

## 1. Overview

The master dataset represents a consolidated history of all international football matches since **1970-01-01** (the modern era of football). 

Each row in the dataset represents **one match**, containing only information that was available **before kickoff**. All post-match outcomes (like the actual scores) are strictly isolated as prediction labels, not features.

## 3. Schema & Data Dictionary

### Match Metadata & Labels (Target Variables)
| Column Name | Type | Description |
|---|---|---|
| `date` | `datetime64[ns]` | Match date |
| `home_team` | `object` (string) | Harmonized home team name |
| `away_team` | `object` (string) | Harmonized away team name |
| `home_score` | `float64` | Home team goals scored (Target/Label) |
| `away_score` | `float64` | Away team goals scored (Target/Label) |
| `tournament` | `object` | Tournament or competition name |
| `city` | `object` | Match city |
| `country` | `object` | Match country |
| `neutral` | `bool` | Whether the match was played on a neutral venue |

### Pre-Match Strength & Rating Features
| Column Name | Type | Description |
|---|---|---|

| `home_rank` | `float64` | Home team FIFA rank before the match |

| `home_fifa_points` | `float64` | Home team FIFA points before the match |
| `away_fifa_points` | `float64` | Away team FIFA points before the match |

## 4. Design Decisions & Verification
- **Deduplication**: Rows with duplicate primary keys (`date`, `home_team`, `away_team`) are automatically removed, keeping the first occurrence.
- **Dtypes**: Dates are strictly cast to `datetime64[ns]` to enable high-performance temporal joins.
- **Null Handling**: Downstream models must handle null values in ranking columns for matches played before August 1993 (prior to the introduction of FIFA rankings).
